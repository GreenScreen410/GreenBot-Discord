# Gemini Instructions for GreenBot-Discord

이 문서는 GreenBot-Discord 프로젝트에서 AI 어시스턴트(Gemini)가 따라야 할 가이드라인을 정의합니다.

## 🚀 핵심 원칙 (Core Principles)

- **미니멀리즘 (Minimalism)**: 꼭 필요한 기능만 구현하며, 불필요한 추상화나 과도한 인프라는 지양합니다.
- **단순함 (Simplicity)**: 코드는 직관적이어야 하며, 복잡한 로직보다는 명확한 흐름을 선호합니다.
- **실용주의 (Pragmatism)**: 오버엔지니어링을 피하고, 현재의 요구사항을 가장 효율적으로 해결하는 방법을 선택합니다.

## 🎨 코딩 스타일 (Coding Style)

### 1. 명령어 구현 (Commands)

- **참고 모델**: [cat.ts](file:///home/paulj/GreenBot-Discord/src/commands/activity/cat.ts), [ping.ts](file:///home/paulj/GreenBot-Discord/src/commands/system/ping.ts)처럼 간결하고 명확한 구조를 유지합니다.
- **비정형 코드 기피**: `stock` 명령어와 `guesstheflag`는 구조가 복잡하거나 정리가 필요한 상태이므로 스타일 가이드로 참고하지 않습니다.
- **코드 간소화**: 예외 처리를 위한 조건문을 과다하게 사용하지 않아야 합니다. 대부분은 입력값이 이미 검증되었기 때문입니다
- **변수명 사용**: 변수명은 되도록이면 공식 문서와 자료에서 제공하는 형식을 따릅니다

### 2. 기술 스택 및 라이브러리 활용

- **Discord.js**: 최신 버전의 기능을 활용하되, 빌더 패턴을 적절히 사용하여 선언적으로 작성합니다.
- **Drizzle ORM**: 데이터베이스 조작 시 Drizzle을 사용하며, 스키마에 정의된 타입을 엄격히 준수합니다.

### 3. 문서화 및 커뮤니케이션

- 코드는 그 자체로 설명이 가능해야 하며, 주석은 정말 필요한 경우(복잡한 알고리즘, 외부 API 제약 등)에만 최소한으로 작성합니다.
- 변경 사항은 명확하고 간결하게 설명합니다.

## ⚠️ 주의 사항

- 코드를 새로 작성하거나 수정할 때 항상 "이것이 정말 필요한 코드인가?"를 자문합니다.
- 유저의 취향에 맞춰 군더더기 없는 '미니멀'한 코드를 생산하는 데 집중합니다.
